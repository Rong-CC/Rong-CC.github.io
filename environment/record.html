# iview DatePicker 日期限制

提出一个需求就是日期小于最后业务日期且不能大于今天日期

如果是能大于今天就直接用iview 的options就行了

这是官网例子

<template>
    <Row>
        <Col span="12">
            <DatePicker type="date" :options="options3" placeholder="Select date" style="width: 200px"></DatePicker>
        </Col>
    </Row>
</template>
<script>
    export default {
        data () {
            return {
                options3: {
                    disabledDate (date) {
                        return date && date.valueOf() > Date.now() // 日期不能大于今天
                    }
                }
            }
        }
    }
</script>

本来就一个很简单的问题，由于对iview不太了解就直接百度了给出的结果如下:

<script>
    export default {
        data () {
            return {
                options3: {
                    disabledDate (date) {
                       return date.getTime() > Date.now() - 8.64e6 // 日期不能大于今天
                    }
                }
            }
        }
    }
</script>

开始使用以上方法用实现的日期不能大于今天，但是上述的需求就遇到一个坑

因为一个条件是来自于父组件，iview写的disabledDate里面不能获取this，应该是作用域的问题，用箭头函数解决作用域的问题，但是this并没有获取到prop传过来的数据，所以现在在watch 里面直接写条件

# iview 在按回车会刷新页面

​ 因为iview input组件的默认绑定enter，导致在单个input 按回车会提交一个空表单，导致页面会刷新

​ 解决方案

​ 1、在From 表单上写 @submit.native.prevent

​ 2、每个input 上写上@keydown.enter.native.prevent 来阻止他的默认事件
