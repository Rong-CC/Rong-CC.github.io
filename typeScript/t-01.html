function test():void{
  alert('hellow world')
}

声明一个void类型的变量没有什么用，因为你只能将它复制为undefined 和 null

任意值

​ 任意值（Any）用来表示允许赋值为任意类型。

联合类型

​ 表示取值可以为多种类型中的一种

​ 联合类型使用 | 分隔每个类型 下例子允许 myFavorite 的类型 string 或者number，不能为其他类型

let myFavorite : string | number;
myFavorite = "seven";
myFavorite = 7

访问联合类型的属性或方法

当 TypeScript 不确定一个联合类型的变量到底是哪个类型的时候，我们只能访问此联合类型的所有类型里共有的属性或方法：

function getLength(something: string | number): number {
    return something.length;
}
//上例中，length 不是 string 和 number 的共有属性，所以会报错

let random: string| number
random = "seven"
console.log(random.length); // 5
random = 7
console.log(random.length)  // 编译错误

# 对象的类型 ------接口

定义对象的类型使用 interface

TypeScript 中的接口是一个非常灵活的概念，除了可用于对类的一部分行为进行抽象 (opens new window)以外，也常用于对「对象的形状（Shape）」进行描述。

interface Person {
	name: string;
	age: number;
}
let tom:Person ={
	name: 'tome',
	age: 23
}

定义一个接口Person ，接着定义一个变量tom，他的类型是Person，我们就约束了 tom 的形状必须和接口 Person 一致。 接口一般首字母大写。有的编程语言中会建议接口的名称加上 I 前缀 (opens new window)。

定义的变量比接口少一些属性是不允许的、多也是不允许的

interface Person {
    name: string;
    age: number;
}

let tom: Person = {
    name: 'Tom'
};
// 报错

可选属性

​ 有事我们希望不要完全匹配一一样形状，就可以使用可选属性，

​ 下例：tom类型 age 可以不写属性名字， 但是仍然不允许添加未定义的属性：

interface Person {
    name: string;
    age?: number;
}

let tom: Person = {
    name: 'Tom'
};

let tom: Person = {
    name: 'Tom',
    age: 25,
    gender: 'male'
}; //报错，赋值的时候，变量的形状必须和接口的形状保持一致。

任意属性

有时候我们希望一个接口允许有任意的属性，可以使用如下方式

interface Person {
    name: string;
    age?: number;
    [propName: string]: any;
}

let tom: Person = {
    name: 'Tom',
    gender: 'male'
};

使用 [propName: string] 定义了任意属性取 string 类型的值。

keyof 索引查询

interface Eg1 {
  name: string,
  readonly age: number,
}
// T1的类型实则是name | age
type T1 = keyof Eg1

class Eg2 {
  private name: string;
  public readonly age: number;
  protected home: string;
}
// T2实则被约束为 age
// 而name和home不是公有属性，所以不能被keyof获取到
type T2 = keyof Eg2

typeof

typeof 操作符可以用来获取一个变量或对象的类型

interface Person {
    name: string;
    age: number;
}

const sem: Person = { name: "semlinker", age: 30}
type Sem = typeof sem; // type Sem = Person

# 高级类型

# 映射类型

type Readonly<T> = {
  readonly [P in keyof T]: T[P];
};
  • 我们正在定义一个Readonly具有单个类型参数的泛型类型，名为T。
  • 在方括号内，我们使用了keyof运算符 (opens new window)。keyof T将 type 的所有属性名称表示T为字符串文字类型 (opens new window)的联合。
  • 在in方括号内的信号关键字的，我们正在处理一个映射类型。[P in keyof T]: T[P]表示 type 的每个属性P的类型T都应该转换为T[P]. 如果没有readonly修饰符，这将是身份转换。
  • 该类型T[P]是查找类型 (opens new window)。它表示 type 的属性P的类型T。
  • 最后，readonly修饰符指定每个属性都应该转换为只读属性。
  • 理解：P是T对象里面key值之一

# Partial实现原理

Partial 可以快速把某个接口类型中定义的属性变成可选的(Optional)：

type Partial<T> = {
    [P in keyof T]?: T[P];
};

interface IUser {
  name: string
  age: number
  department: string
}
//类型转换结果
type optional = Partial<IUser>

// optional的结果如下
type optional = {
    name?: string | undefined;
    age?: number | undefined;
    department?: string | undefined;
}

# Pick选取类型中指定类型

type Pick<T, K extends keyof T> = {
  [P in K]: T[P];
}

interface User {
  id: number;
  age: number;
  name: string;
};

// 相当于: type PartialUser = { id?: number; age?: number; name?: string; }
type PartialUser = Partial<User>

// 相当于: type PickUser = { id: number; age: number; }
type PickUser = Pick<User, "id" | "age">

# Omit去除类型中某些项

Omit会构造一个除类型K外具有T性质的类型

export interface Contact{
  name: string; // 姓名
  phone?: string; // 手机号
  email: string; // 邮箱
  avatar: string; // 头像
  userid: string; // id
}

去除email字段

export type OmitEmailContact = Omit<Contact, 'email' >;
OmitEmailContact{
  name: string;
  phone?: string; 
  avatar: string;
  userid: string;
}

去除多个字段email、avatar

export type OmitEmailAvatarContact = Omit<Contact, 'email' | 'avatar'>;
OmitEmailAvatarContact {
  name: string;
  phone?: string; 
  userid: string;
}
