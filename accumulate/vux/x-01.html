    # scroller ios 13.0以上滚动回到顶点问题

    ​ 因为vux 不更新scroller组件存在很多问题，在ios13版本的存在一个滚动问题,当快速滚动时存在回到顶部问题

    经过百度查证了是ios13获取transform的结果和老版本不一样

    老版本：
'matrix(1, -2.4492935982947064, 2.4492935982947064, 1, 0, 19.48200035095215)'

//新版本
'matrix(1, -2.4492935982947064e-16, 2.4492935982947064e-16, 1, 0, 19.48200035095215)'

    解决办法：将\node_modules_vux-xscroll@3.1.12@vux-xscroll\build\cmd\simulate-scroll.js下的getScrollTop方法里面的正则表达式替换成下面（/[-\d.*\d*e-\d]+/g ）的就可以了。

     getScrollTop: function() {
    // var transY = window.getComputedStyle(this.container)[transform].match(/[-\d\.*\d*]+/g);
    var transY = window.getComputedStyle(this.container)[transform].match(/[-\d\.*\d*e\-\d]+/g);
    return transY ? Math.round(transY[5]) === 0 ? 0 : -Math.round(transY[5]) : 0;
  }

    # Scroller 切换不初始化问题

    当scroller 切换了类别时，他的滚动不会初始化位置还是停留再之前的位置，

    ​ http://xscroll.github.io/node_modules/xscroll/doc/XScroll.html#scrollTop

    ​ 最好放在nextTick方法里面

    this.$nextTick(() => {
     this.$refs.scroller._xscroll.scrollTop(0) // 滚动位置初始化
  })
